# Code of Conduct

## Purpose

AgentFS is intended to be a serious open-source systems project. Productive collaboration requires directness, respect, and technical honesty.

This code of conduct applies to project spaces including:

- issues
- pull requests
- discussions
- documentation reviews
- other public and private community interactions related to the project

## Expected Behavior

Contributors are expected to:

- communicate respectfully and clearly
- critique ideas, code, and decisions without attacking people
- make technical claims honestly and with evidence where possible
- accept correction without escalating into personal conflict
- keep the project usable for people with different experience levels and backgrounds

## Unacceptable Behavior

The following behavior is not acceptable:

- harassment, threats, or intimidation
- personal attacks or insults
- discriminatory or demeaning language
- doxxing or publishing private information without permission
- repeated bad-faith argumentation that prevents productive collaboration
- deliberate misrepresentation of security, licensing, or technical facts
- spam, trolling, or disruption for its own sake

## Technical Disagreement

Strong disagreement is acceptable. Personal hostility is not.

Examples of acceptable behavior:

- challenging a design choice with concrete reasoning
- rejecting a patch because it weakens trust boundaries
- asking for better tests, better docs, or stronger evidence

Examples of unacceptable behavior:

- mocking contributors instead of addressing the change
- escalating routine review feedback into personal arguments
- dismissing concerns without reading the material

## Enforcement

Project maintainers may take action including:

- asking for behavior to stop
- removing comments or contributions
- closing discussions
- limiting participation
- banning repeat offenders from project spaces

## Reporting

If you experience or witness behavior that violates this policy, report it to:

- `Justinedevs@jstn.site`

Include links, screenshots, or other context if available.

## Maintainer Standard

Maintainers are expected to apply this policy consistently and to separate technical review from personal reaction.

The goal is not to remove disagreement. The goal is to keep disagreement useful.
