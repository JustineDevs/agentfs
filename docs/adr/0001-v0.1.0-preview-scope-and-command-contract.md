# ADR 0001: v0.1.0 Preview Scope and Command Contract

- Status: Accepted
- Phase linkage: Phase 1
- Derived from:
  - `docs/Roadmap.md`
  - `docs/TA.md`
  - `docs/Concept.md`

> [!IMPORTANT]
> This ADR resolves the preview command ambiguity. Executors must not infer extra commands into v0.1.0 preview scope.

## Context

The user requested a preview flow centered on:

```text
afs init -> afs add -> afs match -> afs doctor
```

But the standing roadmap and TA docs currently establish a stronger documented preview surface around:

- `.agent.db`
- `afs init`
- `afs add rule`
- `afs add hook`
- `afs status`
- `afs doctor`

They do not establish `afs match` as part of the preview command contract.

## Decision

The v0.1.0 preview contract is:

### In scope

| Surface | Meaning |
| --- | --- |
| `.agent.db` | canonical local state file |
| `afs init` | initialize repository-local AgentFS state |
| `afs add rule` | create scoped rule records |
| `afs add hook` | create scoped trusted hook records |
| `afs status` | inspect local runtime state |
| `afs doctor` | verify integrity and local setup |

### Out of scope

| Surface | Why excluded from preview contract |
| --- | --- |
| `afs match` | not yet established in the documented roadmap command surface |
| `afs export` | belongs to reflection/compatibility scope beyond the minimal preview contract |
| `afs ingest` | belongs to migration/compatibility scope beyond the minimal preview contract |

## Acceptance Criteria

- preview docs and issues must not imply `afs match` exists in v0.1.0
- preview docs and issues must include `afs status`
- “basic scoped CRUD” is interpreted for preview as `afs add rule` and `afs add hook`, plus visible state inspection and integrity checks

## Consequences

### Positive

- aligns preview claims to existing roadmap truth
- reduces execution ambiguity
- keeps preview narrow and credible

### Negative

- the operator story is narrower than the original aspirational flow
- later versions must explicitly introduce `afs match` if it becomes real

## Execution Impact

- issue bodies for preview work must use this exact command contract
- README and future release notes must not reintroduce excluded commands without source-doc updates
