# Agent Operating Contract

- Follow repository-local instructions first.
- Do not overwrite unrelated user changes.
- Use `npm test` before changing shared API code.
- Keep policy and runtime memory separate.
- Record assumptions in the task summary.
- Avoid touching deployment config unless the task requires it.
- Prefer explicit scopes when adding rules.
- Never disable security checks to make a demo pass.
- Keep CLI examples copy-pasteable.
- Preserve markdown tables and alerts in documentation.
